# 与上游版本的差异

本项目从 [yishi170336/AI-Teaching-Assistant](https://github.com/yishi170336/AI-Teaching-Assistant) 的提交 `a5ff57b00ac341c60bd96e0bc5b48793f8ff7391` 开始继续开发。

上游版本已经提供了可运行的学生问答页面、LangGraph 路由、混合检索、同类题生成、附件上传、Ollama 与云端模型接入等基础能力。本 fork 没有推翻这些实现，而是将项目定位进一步收缩为“面向电路课程解题过程的 AI 助教”，重点改善学生端的过程辅导、课程知识组织和可靠性。

## 主要改进

| 方向 | 上游基线 | 当前实现 |
|---|---|---|
| 本地模型 | 默认 Ollama `qwen3.5:2b` | 默认 LM Studio `qwen/qwen3.5-9b`，同时保留 Ollama 与云端兼容接口 |
| 模型分工 | 主要复用一个模型 | 支持理解、求解、诊断和辅导四种角色分别配置模型 |
| 意图路由 | 答疑或出题二分 | `qa`、`answer`、`quiz`、`chat` 四路语义路由 |
| 解题流程 | 检索后直接生成答案 | 题目理解 → 内部求解 → 步骤诊断 → 分层辅导 |
| 教学控制 | 默认完整推导 | 逐步引导与完整解答分离，支持 L1–L5 提示等级 |
| 题目状态 | 主要依靠近期聊天记录 | 独立持久化题目分析、参考解、诊断和附件蓝图 |
| 会话记忆 | 较短近期窗口 | 默认 24 轮，长度感知压缩最近 20 条消息 |
| 图片理解 | 提取题干和宽泛拓扑 | 按导线、连接点、节点和支路建立拓扑；不完整时拒绝猜测 |
| 检索依据 | 容易在信息不足时返回弱相关片段 | 题意完整性、知识点文本证据和经实际分数标度校准的相关度门控；独立概念答疑自动检索多教材 |
| 知识导入 | 教材清洗和结构化 Excel 题库 | 批量自动分类、试卷 PDF 抽题、OCR 适配、缓存和原子重建 |
| 题目—教材关联 | 无独立关联文件 | 生成 `supported_by` 关系并保留章节、页码和共享知识点 |
| 学习空间 | 最近会话 | 新增错题本、分类管理和课程知识图谱 |
| 知识图谱文本 | 无 | OCR 噪声过滤、来源聚合、基础定义校核卡片、主章节证据评分和关联错题 |
| 公式显示 | 基础 KaTeX | 增加混合 Markdown/LaTeX 容错和未闭合公式修复 |
| 生成超时 | 固定总时限 | 仅在 600 秒仍无第一个可见 token 时中止，输出后不设总时限 |
| 失败恢复 | 重新发送文字 | 同时保留原始附件 ID，避免重试丢失电路图 |

## 路由设计变化

当前路由首先判断“学生想完成什么”，然后再决定是否需要多智能体协作：

- 概念问题、结果追问和省略式上下文问题走直接答疑，避免无意义地重复完整解题链；
- 明确求解、学生步骤检查、验算和错因诊断才调用四智能体子图；
- 只有出现明确命题动作时才进入出题流程，“这道练习题怎么做”不会被误判为出题；
- 寒暄、离题内容、乱码、提示注入和缺少题干的请求进入会话引导，不调用 RAG 或解题节点。

独立知识点答疑会先经过教材检索；“这个答案为什么这样”等上下文追问只有在用户明确要求出处时才追加检索，避免在指代对象尚不明确时展示无关章节。

## 知识库变化

当前导入链将“开发者上传课程资料”视为本地数据处理任务：

1. 对文件分类为教材、试卷或结构化题库；
2. 优先保留章节、页码和公式结构；
3. 扫描 PDF 可使用 macOS Vision OCR，或通过统一接口接 PDF-Extract-Kit、MinerU、Docling；
4. 提取结构化题目并生成教材支持关系；
5. 同时建立向量索引和 BM25 索引；
6. 从知识点、章节、来源和错题记录生成可浏览的知识图谱。

与直接展示原始切片相比，知识图谱节点会过滤页眉页脚、缺图图号、OCR 乱码和不完整公式。若基础定义的 OCR 可靠性不足，可使用人工校核的定义卡片，同时保留原始教材出处供复核。

索引加载前还会检查 `chunks.jsonl`、`index_meta.json` 和 `vectors.faiss` 是否完整；中断构建或切换分支留下的半成品目录会被标记为“需要重建”，不会让服务启动过程抛出文件缺失异常。

## 尚未实现

以下内容不应在项目介绍中标记为已完成：

- ngspice、Xyce、CircuitJS 或其他电路仿真器接入；
- 教师端教材审核、作业发布、班级统计等完整业务；
- 用户登录、课程权限和多租户隔离；
- 生产级数据库、任务队列和容器沙箱；
- 对所有电路图样式都可靠的拓扑识别。
